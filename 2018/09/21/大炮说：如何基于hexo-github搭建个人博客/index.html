大炮说：如何快速搭建免费的个人博客

发表于 2018-09-21 |

搭建个人博客的意义

在现在流行微信公众号的时代下，个人博客的价值是什么呢？
就个人而言，个人博客比微信公众号屏幕更大一些，书写更加自由一些，也不会因为写点什么触犯了规则而被封号的情况。
所以，如果你想要一个自由的发布环境，一个没有广告的地方，一个能够自己掌控内容与发布时间的地方的话，自己搭建一个博客是一个比较推荐的方式。
当然自己搭建的，总觉得比用别人的舒服一些。


搭建的原理

个人博客的配套方案有很多种，本套方案只是其中的一种（特点就是快、而且成本免费）：

  • 你需要掌握编辑工具 Visual Studio Code
  • 你需要有 GitHub账号，你需要知道GitHub里面的 GitHub Pages 功能。
  • 你需要掌握 hexo 博客框架 和 hexo NexT主题包。
  • 你需要掌握 markdown语法。
  • 你需要掌握 git。
  • 你需要掌握 node.js 里面 npm 的包管理。
  • 你需要知晓 图床 概念。

以上内容虽然看起来蛮多的，但是一旦配置好了以后，书写也是相当方便的。 这里面我们要知道博客其实就像一个个人网站一样，每天可以在上面发布自己的各种信息，学习笔记，新的等等。所以首先我们需要一个可以存放我们博客内容的空间，这个也就是 GitHub Pages 的作用了，它可以当做一个云服务器，然后展示我们存放到上面的静态网页。

有了服务器以后，接着还要有静态网页，我们知道静态网页是用html 的标签语言写成的，一般会有标签头和标签尾部，html 网页并不适合直接拿来书写文章，如果用 html 来写的话，你还需要写一个后台管理系统，前后台结合起来，发文章在后台管理系统发，然后前台页面负责展示。【时间成本太大，不推荐】

所以，我们需要使用 hexo 博客框架来搭建博客，她的原理是通过我们在编辑器写文章，然后通过框架来将其转化为 html 网页的格式，然后通过自带的发布系统输入一行命令 hexo d就可以实现将新增或者修改的文章通过 GitHub Pages 发布到网页上了。【另外，她还有各种各样的网页主题包，可以很方便的更换各种博客主题，变化样式也就不用自己重写CSS了】

另外 hexo 博客框架是用 markdown语法 写的，这是一种简化语言，5分钟以内就能掌握。

第一步：我们先下载一个编辑工具 Visual Studio Code,拿来写文章和修改hexo的配置。
第二步：注册GitHub账号，找到 GitHub Pages发布方法。
第三步：下载好 node.js ，配置好 npm 和 cnpm 淘宝镜像。
第四步：通过 npm 下载好 hexo 的脚手架，安装好依赖后，配置好里面的参数。
第五步：接着你需要下载 git,将其关联到 GitHub 上面。
第六步：然后在 hexo 中配置好 GitHub。
第七步：用 markdown语法 来写文章。
第八步：写好文章后用命令语句将其转化为静态网页，然后 hexo d 用发布。

注：不建议没有前端知识的朋友自己搭建博客，但是跟着本篇文章一步一步搭建也是可以的，只要不怕麻烦。
